# ZENTRA — คู่มือติดตั้งและใช้งาน (สำหรับส่งมอบ/รันบนเครื่องอื่น)

เอกสารนี้แบ่งเป็น 3 ส่วน
- **ส่วน A — เตรียมแพ็กเกจก่อนส่ง** (ทำบนเครื่องของคุณ) ← *สำคัญที่สุด อย่าข้าม*
- **ส่วน B — ติดตั้งบนเครื่องปลายทาง**
- **ส่วน C — คู่มือการใช้งาน** (ทีละหน้าจอ)

ZENTRA เป็นแอปเดสก์ท็อป (PyWebView + FastAPI) ที่ห่อ AI engine ไว้ในตัวเดียว ประมวลผล
**ในเครื่องทั้งหมด ไม่มีภาพออกนอกเครื่อง** รันโดยเปิด `app.py` ซึ่งจะ start เว็บเซิร์ฟเวอร์
ที่ `127.0.0.1:7788` แล้วเปิดหน้าต่างแอปเต็มจอ

## ⚠️ ส่วน A — เตรียมแพ็กเกจก่อนส่ง (ทำบนเครื่องคุณก่อน zip)

โฟลเดอร์ที่ต้องส่งคือ `Application/ZENTRA_application-main/` ทั้งโฟลเดอร์
แต่มีไฟล์ที่ **git ไม่เก็บ** (gitignored) และ **จุดที่ต้องแก้** ก่อน มิฉะนั้นเครื่องปลายทางจะรันไม่ได้

### A1. ต้องใส่ไฟล์โมเดลไปด้วย (git ไม่เก็บให้)
โมเดลอยู่ใน `backend/models/` และถูก gitignore → **การ clone/zip ผ่าน git จะไม่มีไฟล์เหล่านี้**
ต้องคัดลอกไปด้วยมือ:

| ไฟล์ | จำเป็น | หน้าที่ |
|---|---|---|
| `backend/models/person_v2.pt` | **ใช่** | ตรวจจับ+ติดตามคน (รวมคนที่ล้มนอน) — หัวใจของทุกโมดูล |
| `backend/models/yolo11n-pose.pt` | **ใช่** | keypoints สำหรับตรวจการล้ม |
| `backend/models/yolo11s.pt` | แนะนำ | โมเดลสำรอง (ดาวน์โหลดเองได้ถ้าเน็ตมี) |
| `backend/models/ppe_finetuned.pt` | *ยังไม่มี* | ตรวจ PPE — ถ้าไม่มี โมดูล PPE จะ **พักอัตโนมัติ** (person/zone/fall ยังทำงาน) |

> ถ้าเครื่องปลายทาง **ไม่มีเน็ต** ต้องใส่ครบทั้ง 3 ไฟล์แรก (ตัว person/pose ปกติดาวน์โหลดเองได้ แต่ออฟไลน์ทำไม่ได้)

### A2. 🔴 แก้ path โมเดลให้พกพาได้ (ship-blocker)
ตอนนี้ `backend/.env` บรรทัด `PERSON_MODEL` ชี้เป็น path เต็มของเครื่องคุณ:
```
PERSON_MODEL=D:/NSC26_Zentra/Application/ZENTRA_application-main/backend/models/person_v2.pt
```
เครื่องอื่นไม่มี path นี้ → จะโหลดโมเดลผิด (ตกไปใช้ yolo11s ที่มองไม่เห็นคนล้ม) **เลือกทำอย่างใดอย่างหนึ่ง:**

- **วิธีแนะนำ (แก้ครั้งเดียว จบ):** แก้ `backend/config.py` ให้ default เป็น person_v2 อัตโนมัติ แล้ว *ลบ/คอมเมนต์* บรรทัด `PERSON_MODEL` ใน `.env` ทิ้ง:
  ```python
  # ใน config.py แทนบรรทัด PERSON_MODEL เดิม
  _person_default = MODELS_DIR / "person_v2.pt"
  if not _person_default.exists():
      _person_default = MODELS_DIR / "yolo11s.pt"
  PERSON_MODEL = os.getenv("PERSON_MODEL", str(_person_default))
  ```
  วิธีนี้ config จะคำนวณ path ที่ถูกต้องบนทุกเครื่องเอง ตราบใดที่ไฟล์ `person_v2.pt` อยู่ใน `backend/models/`
- **วิธีเร็ว (ต่อเครื่อง):** ให้ผู้ติดตั้งแก้บรรทัดนี้ใน `.env` เป็น path เต็มบนเครื่องนั้น

### A3. 🔴 เพิ่มไฟล์ Demo (ship-blocker)
ปุ่ม **"เข้าโหมด Demo"** เรียกไฟล์ `data/demo.mp4` ซึ่ง **ยังไม่มี** → ปุ่มจะพัง
คัดลอกคลิปตัวอย่างมาวางเป็นไฟล์นี้ (มีคลิปให้ที่ `backend/data/demo_clips/`):
```
copy backend\data\demo_clips\fall-02-cam0-demo.mp4  data\demo.mp4
```

### A4. ล้างไฟล์ตั้งค่าเดิม (ความลับ + ให้โมดูลทำงานครบ)
`data/settings.json` ปัจจุบันมี **LINE token จริง**, group id ที่พิมพ์ผิด (มี `"` เกิน) และตั้ง
`cameras.cam0.roles = ["fall"]` (รันแค่ fall โมดูลเดียว) — **ไม่ควรส่งไฟล์นี้ไป**
ทางที่ดีที่สุด: **ลบ `data/settings.json` ทิ้ง** ก่อน zip → เครื่องปลายทางจะสร้างค่าเริ่มต้นใหม่
(ทุกโมดูลเปิด, ไม่มี token) ให้เอง แล้วผู้ใช้ค่อยตั้งค่า LINE ใหม่ในหน้า ตั้งค่า

### A5. ใส่ไฟล์ `.env` (git ไม่เก็บ)
`backend/.env` ก็ถูก gitignore เช่นกัน คัดลอกจาก `backend/.env.example` แล้วปรับ โดยเฉพาะ:
```
# เครื่อง CPU ล้วน (ไม่มีการ์ดจอ NVIDIA) — สำคัญมากสำหรับ fall:
PPE_INFER_DEVICE=cpu
FALL_MODE=yolo          # โหมด rule-only ที่ทำงานได้จริงบน CPU (hybrid จะไม่ยิงเลยบน CPU)
PERSON_IMGSZ=640
FALL_POSE_IMGSZ=640
FALL_LOOP_FPS=10
STREAM_FPS=10
# เครื่องที่มี GPU NVIDIA → PPE_INFER_DEVICE=cuda และใช้ FALL_MODE=hybrid ได้
```
> ทำไมต้องตั้ง `FALL_MODE=yolo` บน CPU: โมเดลล้มแบบ Transformer ต้องการ 30 เฟรมต่อเนื่อง
> ซึ่งบน CPU เก็บไม่ทัน → ค่า p_fall จะเป็น 0 ตลอด ระบบเลยต้องใช้ชั้น "กฎฟิสิกส์" (yolo mode) แทน

### A6. เลือกวิธีแพ็ก Python
มี 3 แนว เรียงจากง่าย→พกพาสุด:

| วิธี | เหมาะกับ | ข้อดี/ข้อเสีย |
|---|---|---|
| **1. โค้ด + ให้เขาลง Python เอง** | ผู้รับพอมีความรู้ | ง่ายสุด แต่เครื่องปลายทางต้องมี Python 3.11 + เน็ตครั้งแรก (โหลด torch ~2 GB) |
| **2. แนบ `.venv` ไปด้วย** | เครื่องออฟไลน์/ผู้รับไม่ถนัด | ไม่ต้อง pip install แต่ต้องเป็น Windows + สถาปัตย์เดียวกัน และไฟล์ใหญ่ (~2 GB) |
| **3. สร้าง .exe ด้วย PyInstaller** | ส่งกรรมการ NSC | ดับเบิลคลิกรันได้เลย ไม่ต้องมี Python แต่ตั้งค่ายากสุด ต้องทดสอบเยอะ |

> ก่อนแพ็ก ให้ลบของหนักที่ไม่ต้องส่ง: `.venv/` (ถ้าใช้วิธี 1), `**/__pycache__/`, `runs/`,
> `backend/archive/`, และ `backend/yolo11l/m/x*.pt` (โมเดลที่ไม่ได้ใช้)

**ตรึงเวอร์ชัน dependency (แนะนำ):** บนเครื่องคุณรัน `pip freeze > requirements.lock.txt` แล้วแนบไปด้วย
เพื่อให้เครื่องปลายทางได้เวอร์ชันเดียวกันเป๊ะ (กัน mediapipe/protobuf ตีกัน)

## ส่วน B — ติดตั้งบนเครื่องปลายทาง (Windows)

### B0. ความต้องการเครื่อง
- **Windows 10/11** (64-bit)
- **Python 3.11** — [python.org/downloads](https://www.python.org/downloads/release/python-3119/)
  ตอนติดตั้ง **ติ๊ก "Add python.exe to PATH"**
- **Microsoft Edge WebView2 Runtime** — จำเป็นสำหรับหน้าต่างแอป
  (Win11 มีมาให้แล้ว; ถ้าเปิดแอปแล้วหน้าต่างว่าง/ไม่ขึ้น ให้ลง "Evergreen WebView2 Runtime" จากเว็บ Microsoft)
- **เน็ต** — ครั้งแรกเพื่อ `pip install` (~2 GB) และเพื่อกราฟกิจกรรมบนแดชบอร์ด (Chart.js โหลดจาก CDN;
  ถ้าไม่มีเน็ตกราฟจะขึ้นข้อความแทน ส่วนอื่นทำงานปกติ)
- **กล้อง** USB webcam หรือ IP camera (RTSP) — หรือใช้ไฟล์วิดีโอ/Demo ก็ได้
- RAM ≥ 8 GB, ยิ่ง CPU แรงยิ่งลื่น (มีการ์ดจอ NVIDIA จะเร็วขึ้นมาก)

### B1. คัดลอกโฟลเดอร์
วางโฟลเดอร์ `ZENTRA_application-main` ไว้ที่ใดก็ได้ เช่น `C:\ZENTRA\` (เลี่ยง path ที่มีอักษรไทย/เว้นวรรค)

### B2. สร้าง virtualenv + ติดตั้ง dependency
เปิด **PowerShell** ในโฟลเดอร์นั้น แล้วรัน:
```powershell
py -3.11 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements.txt
pip install -r backend\requirements.txt
```
> ถ้าแนบ `.venv` มาแล้ว (วิธี A6-2) ข้ามขั้นนี้ได้เลย

### B3. ตรวจไฟล์ที่ต้องมี
- `backend/models/person_v2.pt` และ `yolo11n-pose.pt` อยู่ครบ
- `backend/.env` มีอยู่ (ตั้ง `PPE_INFER_DEVICE=cpu`, `FALL_MODE=yolo` ถ้าไม่มีการ์ดจอ)
- `data/demo.mp4` มีอยู่ (ถ้าจะใช้ปุ่ม Demo)

### B4. รันแอป
- **วิธีง่าย:** ดับเบิลคลิก **`run_zentra.bat`** (จะหา `.venv` ให้เอง)
- **หรือสั่งเอง:**
  ```powershell
  .\.venv\Scripts\Activate.ps1
  python app.py
  ```
หน้าต่างแอปจะเปิดเต็มจอ ถ้าอยากออกจากเต็มจอกด **F11** หรือปุ่มปิดมุมขวาบนในแอป

### B5. (ทางเลือก) รันบน Linux ด้วย Docker
เหมาะกับกล้อง RTSP บนเซิร์ฟเวอร์ (Docker บน Windows/Mac เข้าถึง USB webcam ไม่ได้):
```bash
docker compose up -d      # เปิดที่ http://127.0.0.1:7788/ (ไม่มีหน้าต่างเดสก์ท็อป ใช้ผ่านเบราว์เซอร์)
```

## ส่วน C — คู่มือการใช้งาน

### ผังการใช้งาน
```
Splash (โหลด) → เลือกแหล่งภาพ → แดชบอร์ด
                                   ├─ กล้อง (Cameras)
                                   ├─ แก้ไขพื้นที่ (Zone)
                                   ├─ ประวัติ (History)
                                   └─ ตั้งค่า (Settings)
```

### C1. หน้าเลือกแหล่งภาพ (Source)
เปิดแอปมาจะให้เลือกแหล่งภาพ:
- **Webcam (USB)** — กล้องที่เสียบเครื่อง (ปกติเลือกอันนี้)
- **RTSP Stream** — กล้องวงจรปิด IP ใส่ URL เช่น `rtsp://user:pass@192.168.1.100:554/stream1`
- **ไฟล์คอมพิวเตอร์** — เลือกไฟล์วิดีโอมาทดสอบ
- **▶ เข้าโหมด Demo** — ใช้คลิปตัวอย่าง ไม่ต้องต่อกล้อง (เหมาะกับการสาธิต)

กด **เชื่อมต่อ** → เข้าสู่แดชบอร์ด ถ้าเปิดกล้องไม่ได้จะขึ้นข้อความแดงบอกสาเหตุ

### C2. แดชบอร์ด (Dashboard) — หน้าหลัก
- **แถบ KPI ด้านบน:** การละเมิดวันนี้ · เวลาทำงาน · กล้องออนไลน์ · โมดูล AI (x/3 ที่ทำงาน)
- **คะแนนความปลอดภัย** (วงกลม): 100 ลบตามจำนวนเหตุการณ์ที่ยืนยันแล้ว เทียบเมื่อวาน/สัปดาห์นี้
- **ภาพสด (LIVE):** วิดีโอพร้อมกรอบตรวจจับ
  - กรอบ **เขียว** = คน + ใส่ PPE ครบ · **ฟ้า** = กำลังตรวจสอบ/ยังไม่รู้ · **แดง** = ละเมิด/ล้ม
- **สถานะโมดูล:** จุดสี เขียว=ทำงาน เทา=สแตนด์บาย/ปิด แดง=ผิดพลาด
- **การแจ้งเตือนล่าสุด** + toast เด้งเมื่อมีเหตุ
- ⚠️ ถ้าขึ้นแถบแดงล่างจอ **"ระบบ AI ไม่ทำงาน"** = เอนจินโหลดไม่สำเร็จ (มักเพราะไม่มีไฟล์โมเดล)
  — ภาพจะไม่มีกรอบเลย **อย่าเข้าใจว่า "ปลอดภัยดี"**

### C3. กล้อง (Cameras) — ตั้งค่าต่อกล้อง
กำหนดได้ว่าแต่ละกล้องเปิดโมดูลอะไรบ้าง (บทบาท/roles): **PPE · โซน · การล้ม** และเลือกว่า PPE
จะบังคับใส่อะไรบ้าง (หมวก/เสื้อกั๊ก/…)
> ข้อควรระวัง: ถ้าเอาโมดูลที่วาดกรอบคน (zone/fall/ppe) ออกหมด **ภาพจะไม่มีกรอบเลย**
> ค่าเริ่มต้น (ไม่ตั้งอะไร) = เปิดทุกโมดูล

### C4. แก้ไขพื้นที่ (Zone Editor) — วาดเขตอันตราย
1. **ต่อกล้องก่อน** (ไม่งั้นพื้นหลังเป็นจอมืด)
2. เลือกกล้องจากดรอปดาวน์มุมขวาบน
3. **คลิกวางจุด** ไปเรื่อย ๆ ให้เป็นรูปหลายเหลี่ยมคลุมพื้นที่อันตราย · ลากมุมเพื่อปรับ · ปุ่ม **ย้อน/ล้าง**
4. กด **บันทึกพื้นที่**
- ระบบจะเตือน (ระดับ **alert**) เมื่อ **จุดเท้า** ของคนเข้าไปในเขต และยืนยันต่อเนื่องพอ
- มี 2 ชนิด: **danger** (เขตอันตราย = เตือน) และ **exclusion** (เขตยกเว้น = ไม่ตรวจคนในนั้น เช่นห้องคุมงาน)

### C5. ประวัติ (History)
- ดูเหตุการณ์ย้อนหลัง กรองตามวัน/ช่วงวัน มีภาพหลักฐานของแต่ละเหตุการณ์
- **ส่งออก CSV** และ **ดาวน์โหลดรายงาน PDF** (รายงานความปลอดภัยแบบทางการ มีช่องเซ็นชื่อ จป.)
- ปุ่ม **ล้างประวัติ** ลบเหตุการณ์+ภาพทั้งหมด (ใช้ตามหลัก PDPA)
- คอลัมน์ `line_sent` บอกตรง ๆ ว่าส่ง LINE สำเร็จหรือไม่

### C6. ตั้งค่า (Settings)
- **การแจ้งเตือน LINE:** ใส่ Channel Access Token + Group ID 3 ระดับ (หัวหน้างาน/Safety/ฉุกเฉิน)
  - warning(PPE)→หัวหน้างาน · alert(โซน)→Safety+หัวหน้า · emergency(ล้ม)→ทุกกลุ่ม
  - เว้นว่าง = ไม่ส่ง LINE (เหตุการณ์ยังขึ้นในแอป+History ปกติ)
  - Token จะไม่ถูกแสดงกลับหลังบันทึก (เก็บเป็นความลับ)
- **AI:** ความเชื่อมั่น PPE, เกณฑ์การล้ม, จำนวนเฟรมยืนยัน
- **การแจ้งเตือน:** ระยะ cooldown แต่ละระดับ · เปิด/ปิดรายระดับ · **อัปโหลดภาพขึ้น LINE** (ค่าเริ่มต้น
  ปิดตามหลัก PDPA — เปิดแล้วภาพหลักฐานจะถูกส่งออกไปโฮสต์ภายนอก)
- **ข้อมูล:** เก็บข้อมูลกี่วัน (retention) แล้วลบอัตโนมัติ
- **รายงาน:** ชื่อสถานที่/บริษัท/ผู้จัดทำ ที่จะพิมพ์บนรายงาน PDF

### C7. การแจ้งเตือน LINE (ถ้าจะใช้)
1. สร้าง LINE Official Account + Messaging API ที่ [developers.line.biz](https://developers.line.biz/)
2. เอา **Channel Access Token** มาใส่ในหน้า ตั้งค่า
3. เชิญบอทเข้ากลุ่ม แล้วเอา **Group ID** ของแต่ละกลุ่มมาใส่
4. กดบันทึก แล้วลองให้เกิดเหตุการณ์เพื่อทดสอบ (ดูคอลัมน์ `line_sent` ในประวัติ)
> ⚠️ ถ้าคัดลอก Group ID มา **ระวังอย่าให้มีอักขระ `"` หรือช่องว่างติดมาท้าย** (ของเดิมในเครื่องมีบั๊กนี้
> ทำให้กลุ่ม Safety/ฉุกเฉินส่งไม่ออก)
